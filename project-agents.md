# project-agents.md — ServerRat

ServerRat (serverrat.perezbox3.com) is a no-login Rust server finder whose differentiator is the **population curve** — how a server holds players across a wipe cycle (avg on day 1/2/3/5/7), not just its wipe-day spike. Stack: Node.js 18+ ESM, Express 4, better-sqlite3 (WAL, FK on), vanilla-JS frontend with a hand-rolled SVG sparkline, no build step; it proxies and caches the BattleMetrics free-tier API (A2S is parked for v2) and runs on the shared Linode under PM2 on **port 3003** behind an Apache reverse proxy. The riskiest assumption — that the free tier exposes enough population history to compute the curve — is settled in **Task 1 (a spike, not a feature)** before any product code; `docs/battlemetrics-findings.md` is the resulting data contract and the source of truth for all API endpoints and JSON paths. Build order and definitions of done live in `serverrat.md`; per-agent guidance: **tech-lead** runs the mandatory re-plans after Task 1 and Task 4 and keeps scope honest against the Out-of-Scope/Parked lists; **code-reviewer** gates every task; **security-reviewer** gates any task touching outbound requests, the `BATTLEMETRICS_TOKEN`, or input handling (Tasks 4, 6, 8); **diagnostic-engineer** is for 30-min-plus blocks with no new observation; **senior-dev-mentor** for design forks. Always pass agents the project path, the current task from `serverrat.md`, and this file. Deploys are run only by Anthony in his persistent Linode SSH session, with his explicit go-ahead — never from Claude Code.
